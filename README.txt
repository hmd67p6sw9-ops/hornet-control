Hornet Control v1.3.4 — автоматичний ID STARLINK

Що змінено:
- користувач більше не вводить ID STARLINK;
- система знаходить найбільший зайнятий номер;
- новому STARLINK автоматично пропонується наступний ID;
- підтримуються наявні формати MINI_001 і MINI-001;
- нові ID створюються у форматі MINI_001;
- поле ID у формі доступне лише для перегляду;
- остаточний ID повторно визначається на сервері;
- два одночасні запити не отримають однаковий ID;
- неможливо створити два STARLINK з однаковим серійним номером;
- порожній серійний номер дозволений;
- перевірка серійника не залежить від регістру;
- автоматичний ID борта та всі попередні функції збережено.

Приклад:
- останній найбільший ID MINI_003;
- система запропонує MINI_004;
- якщо існує MINI_099, наступним буде MINI_100.
